While writing code for your project, we often hit a roadblock, when we know what we want to achieve but are unaware if a function to carry that task already exists. We quickly go to the search bar of our choice and ask what we are looking for. Almost always stackoverflow is at the top of the search and some expert’s code is right there to rescue us. We get delighted, take a note somewhere (my favorite’s are stickies) to remember that for future and continue with our project. Next time, we want to use the same function, many of us have already forgotten but we remember it was written on a sticky. And we spend as much time looking for that sticky (and sometimes more!) as we would have again asked google.

Wouldn’t it be nice if we could practice the functions or any important information for that matter, so we can recall it with ease. Some of you might be lucky and remember the first time you read something. For the rest of us, we need deliberate practice. I have created a package, quizme that can help you with that practice. You can build your personal quiz repository by adding questions and answers right from R console. When you want to practice, you can do that also right from the R console!

How do I get the package?

Package is currently available on Github. You can download it using:

library(devtools)
install_github("pchhina/quizme")
library(quizme)

How do I add questions?

Questions can be added with addq(). At the start of the session however, you have to use the function make_quiz(). This loads the previously created QA list or create blank data structures so new questions can be added.

make_quiz()

With addq(), you can add questions in a single line. Answer must be added in a new line. Answer can consists of multi-lines. Carriage return in a blank line will add your question-answer pair to an R object.

addq()

How do I take quiz?

You would use ask() and a randomly choosen question will be presented at the R console:

ask()
#> NA

How do I look for the answer?

The answer to the question that was presented last can be seen using tell() function:

tell()

What do I do when I am done practicing?

Simply call bye(). This writes the updated QA data to a file in a directory that was created when make_quiz() was called. After that, it also removes the data structures related to testing. And you are done! Happy learning R!
